# 06 – Buddy Style Guide

## Document Status

**Status:** Design Milestone – Approved Foundation  
**Applies To:** CompareBuddy Website and future CompareBuddy brand materials

This document defines the approved use, personality, appearance, placement, and asset management rules for Buddy.

Buddy is the official CompareBuddy Guide.

All use of Buddy must align with:

- 01 – CompareBuddy Design & Brand Guide
- 02 – Website Blueprint
- 03 – Design System
- 04 – Homepage Master Design

---

# 1. Buddy's Role

Buddy is the official CompareBuddy Guide.

Buddy's purpose is to:

- Welcome visitors.
- Make the CompareBuddy experience approachable.
- Guide users through information.
- Support understanding of the product.
- Reinforce the CompareBuddy brand.
- Help users feel confident when using CompareBuddy.

Buddy is not merely decorative.

Every appearance of Buddy should have a clear purpose.

Buddy should make people smile without making CompareBuddy feel less professional.

---

# 2. Buddy's Personality

Buddy is:

- Friendly
- Helpful
- Professional
- Knowledgeable
- Confident
- Approachable
- Reassuring

Buddy should feel like a knowledgeable guide who helps users understand CompareBuddy.

Buddy is not:

- Childish
- Gimmicky
- Silly
- Loud
- Distracting
- Overly exaggerated
- Unprofessional

Buddy must never make CompareBuddy feel like a children's product or a novelty brand.

---

# 3. Official Buddy Appearance

The official Buddy appearance is defined by the approved Buddy artwork in the CompareBuddy asset library.

Buddy's approved visual characteristics include:

- Grey fur
- Green cap
- Blue gloves
- Pink nose
- One blue eye
- One green eye
- Dark grey feet
- Friendly facial expression
- Professional and approachable appearance

These characteristics must remain consistent throughout the CompareBuddy ecosystem.

Buddy must always be recognisable as the same character.

5. Master Buddy Artwork

buddy-official.png is the primary reference artwork for Buddy.

It represents the approved appearance of the CompareBuddy character.

The master artwork must not be casually overwritten, altered, recoloured, or replaced.

If a new official version is approved in the future, it must be saved as a new version rather than silently replacing the original.

Example:

buddy-official-2026.png

This preserves the original approved artwork and provides a reliable reference for future brand development.

6. Approved Buddy Usage

Buddy should be used selectively.

The website should not place Buddy in every section.

The software, product interface, content, and user experience must remain the primary focus.

Buddy should appear when he improves the user's understanding or emotional experience.

Before using Buddy, ask:

Does Buddy improve the experience here?

If the answer is no, Buddy should not be included.

10. Buddy Size Rules

Buddy must always be displayed proportionally.

Buddy must never be:

Stretched
Squashed
Distorted
Cropped in a way that damages the character

Recommended display sizes:

Hero

Approximately:

320–420 px high

The final size depends on the overall layout.

Supporting Sections

Approximately:

180–280 px high

Buddy should support the section without overpowering the content.

Small Supporting Use

Approximately:

80–160 px high

Used for:

Smaller supporting elements
Compact content blocks
Footer usage

These values are guidelines, not rigid requirements.

Visual balance always takes priority.

11. Clear Space

Buddy must have sufficient space around the artwork.

Text, buttons, cards, and other visual elements must not crowd Buddy.

As a general guideline, maintain clear space around Buddy of approximately:

At least one Buddy head width where practical.

Buddy should never appear visually trapped inside a layout.

Do not place important text directly over Buddy.

Do not allow UI elements to visually collide with Buddy.

13. Colour Rules

Buddy must not be casually recoloured.

The approved Buddy artwork is the colour reference.

Do not change:

Fur colour
Cap colour
Eye colours
Glove colours
Nose colour
Foot colours

The approved colour characteristics must remain consistent.

Any deliberate change to Buddy's appearance requires approval as a new brand decision.

14. Prohibited Alterations

Do not:

Change Buddy's clothes.
Change the colour of Buddy's cap.
Change Buddy's eye colours.
Change Buddy's face.
Change Buddy's gloves.
Change Buddy's proportions.
Stretch Buddy.
Squash Buddy.
Recolour Buddy.
Replace Buddy with an unrelated character.
Replace Buddy with an AI-generated alternative.
Turn Buddy into a different visual style without approval.
Make Buddy childish.
Make Buddy gimmicky.
Add random accessories.
Add random objects without a clear purpose.
Use unapproved Buddy artwork.

Buddy must remain recognisable as the official CompareBuddy character.

15. Buddy and Props

Buddy may interact with an object when the object supports the communication purpose of the section.

Appropriate examples may include:

Laptop
Report
Document
Approved product-related visual

Props must be:

Relevant
Professional
Appropriate to the context

Do not add random props simply to make Buddy appear more entertaining.

16. Animation Rules

Buddy animations must be subtle and professional.

Potentially appropriate animations include:

Gentle fade-in
Subtle slide-in
Small entrance movement
Very subtle bounce
Gentle wave
Minimal eye movement or blink, where technically appropriate

Animations must never be:

Excessive
Distracting
Repetitive
Childish
Gimmicky

Buddy should never compete with the user's ability to read or use the website.

Animation should support the experience, not become the experience.

17. Image Quality

Buddy artwork must be suitable for professional digital use.

Approved assets should be:

High resolution
Clear
Properly proportioned
Suitable for their intended display size

Transparent-background assets should be used where appropriate for the design.

Artwork must not be artificially altered in a way that changes Buddy's approved appearance.

The final website must not display visibly pixelated, stretched, or low-quality Buddy artwork.

18. Asset Naming Rules

Buddy asset filenames should be:

Descriptive
Consistent
Lowercase where practical
Easy to identify

Recommended naming format:

buddy-[description].png

Examples:

buddy-report.png
buddy-thinking.png
buddy-wave.png
buddy-laptop.png
buddy-point-left.png

Avoid filenames such as:

IMG_1234.png
final-final.png
new mascot.png
buddy image 2.png

Clear filenames make the asset library easier to manage.

19. Asset Management Rules

The assets/buddy/ folder is the official Buddy asset library.

Do not place random or unapproved Buddy artwork in this folder.

Before a new Buddy image is used:

The artwork must be reviewed.
The appearance must be checked against the approved Buddy characteristics.
The intended use must be defined.
The asset must be given a clear filename.
The asset must be approved before website implementation.
20. Buddy and the Website Header

Buddy should not normally appear in the primary website navigation bar.

The navigation should remain:

Clean
Compact
Professional
Easy to use

Buddy should appear in larger visual areas where he can provide meaningful guidance.

The logo and navigation should remain the primary focus of the header.

21. Buddy and the Footer

Buddy may appear in the footer as a small, subtle brand reinforcement.

The footer should remain professional and uncluttered.

Buddy must not make the footer appear childish or overly decorative.

A small approved Buddy asset may be used where it supports the overall brand experience.

22. Buddy Across the CompareBuddy Ecosystem

Buddy is not limited to the website.

Future uses may include:

Product onboarding
Help and guidance
Empty states
Success messages
Educational content
Marketing materials
Email communications
Product documentation

All future use must remain consistent with this Style Guide.

The same Buddy character should be recognisable across every CompareBuddy touchpoint.

23. New Buddy Artwork Approval

No new Buddy artwork may be used on the website or in official CompareBuddy materials until it has been reviewed and approved.

The approval process should consider:

Does Buddy still look like the official Buddy?
Are the colours correct?
Is the personality correct?
Is the artwork professional?
Does the pose serve a clear purpose?
Does it fit the CompareBuddy brand?
Would a premium software company proudly publish this?

If the answer is no, the artwork should not be approved.

24. Quality Standard

Every use of Buddy must be evaluated against the following question:

Does Buddy improve the experience here?

If Buddy does not improve the user's understanding, confidence, or experience, Buddy should not be included.

Buddy should make people smile without making the software feel less professional.

25. Final Brand Rule

Buddy is the official CompareBuddy Guide.

He is:

Friendly, but not childish.
Helpful, but not distracting.
Professional, but not cold.
Knowledgeable, but not intimidating.
Approachable, but not gimmicky.

Buddy should guide the user.

The product should remain the hero.

The CompareBuddy brand should always remain professional.

Quality Standard:

Would a premium software company proudly publish this?

If the answer is no, improve it before approval.
